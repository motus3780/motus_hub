# 추가 수정 지시서 (C 그룹) — 메인 페이지 레이아웃 재정렬

이 파일은 Claude Code가 참고하는 추가 작업 지시서입니다.
대상 파일: `src/templates/pages.tsx` (renderMainPage 함수)

## 공통 주의사항
- DB 스키마, 멱등성 로직, 발송 로직은 건드리지 말 것
- 섹션을 삭제/이동하면 연결된 클라이언트 JS(`public/static/app.js`)의 이벤트 핸들러가 깨지지 않는지 확인
- 수정 후 `npm run build`가 에러 없이 통과하는지 확인

---

## C-1. "첫 위클리 발행을 준비 중입니다" BETA 박스 삭제
- 파일: `src/templates/pages.tsx`
- 위치: 약 130~148번째 줄, `weeklyBlockHtml`의 베타 모드 폴백 부분
  - `<section class="weekly-beta-fallback" ...>` 블록 (제목 "🌱 첫 위클리 발행을 준비 중입니다", BETA 뱃지, 첫 발행 예정일 안내 포함)
- 요청: 이 베타 폴백 섹션을 노출하지 않도록 한다.
  - 위클리 호가 아직 없을 때(베타 모드) 이 박스 전체를 출력하지 않게 처리.
  - `weeklyBlockHtml`을 빈 문자열('')로 두거나, 베타 폴백 `<section>` 자체를 제거.
  - 단, 정상 위클리 호가 있을 때의 `weeklyBlockHtml`(약 66번째 줄 분기)은 그대로 유지.

## C-2. 구독 카드("매주 월요일 아침, 메일로 받아보기")를 최하단으로 이동
- 파일: `src/templates/pages.tsx`
- 위치: 약 191번째 줄, `<section class="subscribe-card">` 블록 (제목 "📬 매주 월요일 아침, 메일로 받아보기", 이름/이메일 입력 + 구독 신청 폼 포함)
- 요청: 이 구독 카드 `<section>` 블록을 `<main>`의 가장 마지막으로 이동.
  - 폼 id(`sub-form`, `sub-name`, `sub-email`, `sub-btn`)는 그대로 유지해서 app.js의 구독 처리 로직이 계속 동작하게 할 것.

## C-3. 콘텐츠 순서 변경 — 일간 누적 뉴스(+오늘의 핵심 이슈 TOP 3)를 원본 뉴스보다 먼저
- 파일: `src/templates/pages.tsx`
- 현재 `<main>` 내부 순서: 자사 콘텐츠 → (위클리 블록) → 구독 카드 → 일간 누적 뉴스(`daily-supplement`) → 원본 뉴스 → ...
- 요청: **일간 누적 뉴스 섹션(`<section class="card daily-supplement">`, 약 202번째 줄, "📑 일간 누적 뉴스" + summary-content 포함)이 원본 뉴스 섹션(`<!-- 원본 뉴스 -->`, 약 216번째 줄)보다 먼저** 나오게 한다.
  - 일간 누적 뉴스 안의 AI 요약(summary-content)에 "오늘의 핵심 이슈 TOP 3"가 포함되므로, 이 섹션이 원본 뉴스 위에 오면 요구사항 충족.
  - 즉 최종 본문 순서: (자사 콘텐츠) → 일간 누적 뉴스(오늘의 핵심 이슈 TOP 3 포함) → 원본 뉴스 → ... → (맨 아래) 구독 카드
  - C-2의 구독 카드 최하단 이동과 함께 적용.

## 최종 목표 레이아웃 (위에서 아래로)
1. (있으면) 자사 콘텐츠 / 정상 위클리 호 블록
2. 일간 누적 뉴스 (오늘의 핵심 이슈 TOP 3 포함)
3. 원본 뉴스 (카테고리 필터 + 목록)
4. 구독 카드 ("매주 월요일 아침, 메일로 받아보기") ← 최하단
5. 푸터

* 베타 박스("첫 위클리 발행 준비 중")는 더 이상 노출하지 않음.
